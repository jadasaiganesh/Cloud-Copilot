AWS Key Management Service (KMS)

🔐 What is AWS KMS?

AWS KMS is a managed encryption service that lets you easily create and control the cryptographic keys used to protect your data. It integrates seamlessly with most AWS services and supports FIPS 140-2 validated HSMs.

🔒 Real-World Use Case

You're building a healthcare app that stores sensitive patient data in S3 and RDS. With AWS KMS, you can encrypt all that data using customer-managed keys and apply fine-grained access controls via IAM policies.

⚙️ How It Works
  1. Create a Customer Master Key (CMK) in KMS.
  2. Grant IAM roles/users permission to use the key.
  3. Enable encryption in supported services (like S3, EBS, RDS).
  4. Audit key usage via CloudTrail logs.
🌟 Key Features
  • Managed key lifecycle and rotation
  • Automatic integration with AWS services
  • FIPS 140-2 compliance and hardware security modules (HSMs)
  • Envelope encryption for performance
  • Auditability with CloudTrail
  • Custom key store with AWS CloudHSM
💰 Pricing Overview
  • Monthly charge per key: $1
  • Request cost: $0.03 per 10,000 requests
  • Additional fees for HSM-backed custom key stores
🧠 Best Practices
  • Use key rotation to minimize exposure.
  • Apply least-privilege access control on key policies.
  • Monitor key usage with CloudTrail and alarms in CloudWatch.
  • Separate data and key permissions for extra security.
  • Use CMKs for data protection instead of hardcoding encryption.
